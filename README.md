# Chat with Fang

方和与 KaleidScoper 之间的对话导出、点子归档与文件交换仓库。

## 用途

这个项目是 KaleidScoper 和方和之间协作的中转站，主要承担三个角色：

1. **对话内容导出** — 有价值的讨论、跑团记录、长段世界观设定等，整理成 MD 文件归档到 `chats/`。
2. **文件传输** — 方和这边没有 GUI，无法直接上传文件到聊天窗口的场景下，通过 `files/` 中转。
3. **点子沉淀** — 聊天中冒出来的、值得单独拎出来保存的世界观碎片、创作概念、技术思路等，放在 `ideas/`。

## 目录结构

```
Chat-with-Fang/
├── README.md            ← 本文件：总览与使用约定
├── chats/               ← 对话导出
│   └── YYYY-MM-DD/      ← 按日期分目录
│       └── 描述性文件名.md
├── ideas/               ← 独立点子与创作概念
│   └── 描述性文件名.md
├── files/               ← 文件传输中转
│   ├── from-user/       ← 用户 → 方和
│   └── from-fang/       ← 方和 → 用户
└── assets/              ← 图片、视频等媒体资源
```

## 约定

### 命名

- **目录名**：`chats/` 和 `ideas/` 下的文件全部使用**小写英文 + 连字符**命名。
- **日期目录**：`chats/` 下的子目录以 `YYYY-MM-DD` 格式命名，对应对话发生的日期（若跨越多日，取最早的那天）。一天内的多条记录通过在文件名中体现主题区分。
- **文件后缀**：项目内统一使用 `.md`。

### 内容

- **chats/** 下的文件是对话的**直接导出或整理**，保留对话体结构或叙事体均可，但应在文件开头用注释标注原始对话发生的时间段。
- **ideas/** 下的文件是**独立概念文档**，不应该包含对话记录或元信息。它们是一个点子的最终形态，与对话分离。
- **files/** 下的文件视为临时中转——接收方取用后可以删除，不需要长期保留。双方定期清理。

### Commit

- Commit message 用中文或英文均可，但建议清晰说明做了什么（新增、修改、移动）。
- 一个 commit 只做一件事：要么是新增一篇对话导出，要么是更新 README，不要混在一起。

## 内容所有权

本仓库内的所有内容均为 KaleidScoper 与方和在对话中共同创作。KaleidScoper 拥有最终处置权。若需将其中任何内容用于公开项目，由 KaleidScoper 决定标注方式和授权范围。
